(VI) C. Khi bạn cần kết hợp ba lần xác nhận cuối cùng trên nhánh địa phương của mình thành một lần xác nhận duy nhất.

(VI) D. Khi bạn muốn loại bỏ vĩnh viễn mọi sửa đổi không được cam kết trong cả thư mục làm việc và khu vực staging của bạn.

2) What is the primary function of the git reflog command?
(VI) 2) Chức năng chính của lệnh git reflog là gì?

A. To list every remote repository alias configured for the project along with its associated fetch and push URLs.
(VI) A. Để liệt kê mọi bí danh kho lưu trữ từ xa được định cấu hình cho dự án cùng với các URL tìm nạp và đẩy liên quan của nó.
B. To provide a log of where your HEAD (current commit pointer) and branch pointers have been, acting as a safety net to recover "lost" work.
(VI) B. Để cung cấp log về vị trí HEAD (con trỏ cam kết hiện tại) và con trỏ nhánh của bạn, hoạt động như một mạng lưới an toàn để khôi phục công việc "bị mất".
C. To show the complete commit history of the current branch in a paginated log format that can include patch diffs for each commit.
(VI) C. Để hiển thị lịch sử cam kết đầy đủ của nhánh hiện tại ở định dạng log phân trang có thể bao gồm các khác biệt về bản vá cho mỗi cam kết.
D. To prune expired reflog entries and unreachable loose objects from the local Git repository over time so that overall disk usage on the machine is reduced.
(VI) D. Để cắt bỏ các mục nhập lại log đã hết hạn và các đối tượng lỏng lẻo không thể truy cập được từ kho lưu trữ git cục bộ theo thời gian để giảm mức sử dụng đĩa tổng thể trên máy.

Answer: B
(VI) Đáp án: B

11) How does Git Large File Storage (LFS) help manage large binary files?
(VI) 11) git Large File Storage (LFS) giúp quản lý các tệp nhị phân lớn như thế nào?

A. It prevents any individual file whose uncompressed size exceeds one hundred megabytes from ever being added to the repository's object database.
(VI) A. Nó ngăn chặn bất kỳ tệp riêng lẻ nào có kích thước không nén vượt quá một trăm megabyte không bao giờ được thêm vào cơ sở dữ liệu đối tượng của kho lưu trữ.
B. It stores the content of large files on an external server, placing only lightweight pointers in the Git history.
(VI) B. Nó lưu trữ nội dung của các tệp lớn trên máy chủ bên ngoài, chỉ đặt các con trỏ nhẹ trong lịch sử git.
C. It automatically applies lossless compression to binary assets so they occupy less space inside the repository without requiring manual intervention.
(VI) C. Nó tự động áp dụng tính năng nén không mất dữ liệu cho nội dung nhị phân để chúng chiếm ít không gian hơn bên trong kho lưu trữ mà không cần can thiệp thủ công.
D. It converts binary file contents into a human-readable text representation so that Git's standard textual diff tools can compare versions line by line.
(VI) D. Nó chuyển đổi nội dung tệp nhị phân thành dạng biểu diễn văn bản mà con người có thể đọc được để các công cụ tìm khác biệt văn bản tiêu chuẩn của git có thể so sánh các phiên bản theo từng dòng.

Answer: B
(VI) Đáp án: B
