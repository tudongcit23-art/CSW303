4) What is the primary function of a .gitignore file?
(VI) 4) Chức năng chính của tệp .gitignore là gì?

(VI) A. Để liệt kê các tệp mà git nên xóa vĩnh viễn khỏi lịch sử dự án.
B. To specify files and directories that Git should intentionally not track, such as build artifacts or local environment files.
(VI) B. Để chỉ định các tệp và thư mục mà git không nên theo dõi một cách cố ý, chẳng hạn như các tạo phẩm xây dựng hoặc tệp môi trường cục bộ.
C. To mark certain project files as required so that Git always includes them in every commit, even when they have not been modified.
(VI) C. Để đánh dấu các tệp dự án nhất định theo yêu cầu để git luôn đưa chúng vào mọi cam kết, ngay cả khi chúng chưa được sửa đổi.
D. To set up and manage user-wide preferences, credentials, and default behaviors for Git across all repositories on your machine.
(VI) D. Để thiết lập và quản lý các tùy chọn, thông tin xác thực và hành vi mặc định trên toàn người dùng cho git trên tất cả các kho lưu trữ trên máy của bạn.

(VI) Đáp án: B

(VI) A. Họ tạo các bản sao lưu riêng biệt, được mã hóa cho toàn bộ codebase của bạn để có thể khôi phục nếu dữ liệu bị mất.

C. They allow you to work on new features or fixes in isolation without disrupting the stability of the main codebase.
(VI) C. Chúng cho phép bạn làm việc trên các tính năng mới hoặc sửa lỗi một cách riêng biệt mà không làm gián đoạn tính ổn định của codebase chính.

(VI) D. Chúng tự động giải quyết xung đột hợp nhất giữa các nhà phát triển khác nhau.

(VI) A. git get tải lên các thay đổi của bạn, trong khi git pull tải chúng xuống.

(VI) C. git get chỉ hoạt động với nhánh chính, trong khi git pull hoạt động với tất cả các nhánh.
D. git fetch downloads remote updates but does not merge them, while git pull downloads and merges them into your current branch.
(VI) D. git get tải xuống các bản cập nhật từ xa nhưng không hợp nhất chúng, trong khi git pull tải xuống và hợp nhất chúng vào nhánh hiện tại của bạn.
